About Me

This game has been developed by me only for fun. I like retro 8-bit games, and back when I was like 7 years old, I can't afford to get the very popular v-pet device known as the "Digivice" from "Digimon". Actually (Obviously) this game is entirely based on Digimon just to fufill my childhood dream of owning a Digivice.

This project is also all about to practice my programming skills in games related area (sound, animation, etc..). This project is also built entirely with JavaScript(TypeScript) with no framework.

Why?

I am a full-time full-stack dev working in office everyday with frameworks like Vue, Angular, Nestjs, etc... .I just felt kinda overwhelmed by all the frameworks I am working with everyday, so I decided to go back to basic and start from literally scratch. No frameworks, minimum libraries, just TypeScript to keep my code organized. I will make a list of libraries used over here. Oh yea, no canvas has been used in this game!

Also after more than 10 years of gaming experience, I actually felt like I have played everything game and no games interests me anymore. So after working hours, usually my game time is now empty and I felt like I could start a fun little project!

About the game

While this game is all about the nostalgic 8-bit art style. Developing on a modern platform infuse the game with a little modern twist.

Interesting stuffs happened while developing

- Structuring the flow

Hardcoding stuff is generally bad in any type of programming.

- Realizing how convenient frameworks are

After using frameworks like Vuejs, Angular, etc almost everyday. You will soon realize how convenient frameworks are after you start another project from scratch. Things like:

  • 2-way binding
  • Watcher
  • Computed
  • Observable
  • Webpack
  • etc

really makes life easier. While developing this project, mojority of the time I have been thinking "Ah just bind x data with y data" and then realizing I am not using any frameworks at all ;( ... In the end I will end up implementing most of these myself which I learnt a lot from it!

- Developing a Promise listener

This happens when I am developing the dialog section of the game. There's literally arrays of dialogs feed into the Dialog class and will tranverse everything until it reaches the last dialog. Problem is, the Scene controller need to pause the logic and waiting until user has read through all the dialogs. This is where await/async Promise comes into play. Everytime the user starts reading dialogs, there's internal interval set into Dialog class to check for the completed flag. I could just call the method await dialog.untilEnd() and it will pause until user has went through everything.
